## Описание

Проект представляет собой калькулятор для выполнения базовых арифметических операций. Реализован на чистом JavaScript с использованием современных стандартов ES6+ (классы, модули, приватные поля). Интерфейс выполнен в стиле классического настольного калькулятора с сеткой кнопок 4x4.

## Функциональность:

### Какие операции поддерживает?

- Арифметические операции: сложение (+), вычитание (-), умножение (\*), деление (/).
- Ввод цифр: от 0 до 9.
- Десятичная точка: для ввода дробных чисел.
- Очистка: кнопка "C" для сброса текущего выражения.
- Вычисление: кнопка "=" для получения результата.
- Обработка ошибок: вывод сообщения "Ошибка" при некорректном выражении.

### Какие есть ограничения?

- Запрещены два оператора подряд (например, 5++3).
- Оператор не может находиться в конце выражения (например, 5+).
- Точка не может повторяться в одном числе (например, 5.5.3).
- Первым символом выражения не может быть оператор, кроме минуса (для отрицательных чисел).
- Деление на ноль обрабатывается как ошибка.
- Используется функция eval() для вычислений.

### Как обрабатываются ошибки?

- При возникновении ошибки вычисления (деление на ноль, некорректное выражение) на дисплее отображается сообщение "ERROR".
- Калькулятор переводится в состояние `hasError = true`.
- При нажатии любой кнопки, кроме "C", калькулятор автоматически очищается.
- Кнопка "C" сбрасывает состояние ошибки и очищает дисплей.

### класс Calculator

Класс Calculator реализует всю логику работы калькулятора:

| **Метод**                  | **Назначение**                                                |
| :------------------------- | :------------------------------------------------------------ |
| `constructor()`            | Инициализирует дисплей, выражение и состояние ошибки          |
| `init()`                   | Навешивает обработчики событий на кнопки                      |
| `handleButtonClick(value)` | Обрабатывает нажатие кнопки (очистка, вычисление, добавление) |
| `addToExpression(value)`   | Добавляет символ к выражению с валидацией                     |
| `calculate()`              | Вычисляет результат с обработкой ошибок                       |
| `safeEvaluate(expr)`       | Безопасная обертка для eval() с проверкой                     |
| `isValidExpression(expr)`  | Проверяет корректность выражения                              |
| `clear()`                  | Сбрасывает состояние калькулятора                             |
| `updateDisplay()`          | Обновляет дисплей                                             |
| `showError(message)`       | Отображает ошибку                                             |
| `isOperator(char)`         | Проверяет, является ли символ оператором                      |
| `isLastCharOperator()`     | Проверяет, является ли последний символ оператором            |
| `hasDuplicateDot()`        | Проверяет наличие двух точек в текущем числе                  |

### Схема взаимодействия компонентов

1. Пользователь нажимает кнопку в index.html
2. `main.js` ловит событие
3. Вызывается метод `handleButtonClick()` из `calculator.js`
4. `calculator.js` обрабатывает логику (валидация, вычисления)
5. Обновляется `display.value` через `updateDisplay()`
6. `index.html` показывает новый результат на экране

## Логика работы:

### Последовательность действий при нажатии кнопок

1. Пользователь нажимает кнопку (цифра, оператор, точка, C или =)
2. Обработчик события получает текст кнопки
3. Проверка состояния ошибки: если `hasError = true` и нажата не "C", вызывается clear()
4. Анализ кнопки:
   - Если "C" → `clear()`
   - Если "=" → `calculate()`
   - Иначе → `addToExpression(value)`
5. Обновление дисплея через `updateDisplay()`

### Как работает валидация ввода?

В методе `addToExpression()` реализована проверка:

1. Запрет двух операторов подряд
2. Запрет точки в числе, где точка уже есть
   - Валидация: вторую точку просто игнорирует
3. Запрет оператора в начале (кроме минуса)

### Как происходит вычисление?

1. Проверка: `if (!this.expression || this.hasError) return;`
2. Безопасное вычисление: `safeEvaluate(this.expression)`
   - Сначала проверяет выражение через `isValidExpression()`
   - Затем вызывает `eval()` для получения результата
3. Проверка результата: `!isFinite(result)` (деление на ноль, бесконечность)
4. Обновление: результат преобразуется в строку и сохраняется в `expression`
5. Ошибка: при исключении вызывается `showError("Ошибка")`
